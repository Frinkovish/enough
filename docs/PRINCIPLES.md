# Product Principles

1. No guilt — relapses are never punished, shamed, or flagged as failure.
2. Progress over perfection — consistency over streaks; one slip doesn't erase progress.
3. User autonomy — the user always decides; the app suggests, never forces.
4. Behavior science — every feature must be grounded in what actually changes habits, not assumptions or "nice to have" ideas.
5. Small wins — small, achievable actions beat big motivation pushes.

When in doubt, favor simplicity over complexity. A feature that adds friction without a clear behavioral reason gets cut or deferred.
